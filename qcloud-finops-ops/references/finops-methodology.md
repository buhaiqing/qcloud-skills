## 2. 异常检测算法推导

### 2.1 核心思想

异常检测不是单点判断，而是**多维度交叉验证**，避免误报/漏报。

### 2.2 双维度检测（默认实现）

**维度 ii：滚动对比**（业务波动型）
```
ii_ratio = (current - mean(history_3m)) / mean(history_3m)
ii_violated = ii_ratio > 0.20    # 默认阈值 20%
```

**维度 iii：预算对比**（目标导向型）
```
iii_ratio = current / budget.amount
iii_violated = iii_ratio > 0.80    # 默认阈值 80%
```

**置信度规则**：
| ii_violated | iii_violated | 置信度 | 动作 |
|---|---|---|---|
| ❌ | ❌ | NORMAL | 记录，巡检 |
| ✅ | ❌ | MEDIUM | 告警 + 根因分析 |
| ❌ | ✅ | MEDIUM | 告警 + 根因分析 |
| ✅ | ✅ | HIGH | 告警 + 根因分析 + 自动派发巡检 |

### 2.3 维度 i：同期对比（扩展）

适合业务稳定的场景（如传统企业）：
```
i_ratio = (current - same_month_last_year) / same_month_last_year
i_violated = i_ratio > 0.30    # 阈值 30%（同期波动容忍更大）
```

启用方法：在 `example-config.yaml` 中设置 `enable_i: true`。

### 2.4 算法变体

| 变体 | 公式 | 适用 |
|---|---|---|
| **绝对值阈值** | `current > budget * 0.9` | 已有明确预算 |
| **环比 + 同比** | `max(moM, YoY) > threshold` | 业务稳定型 |
| **Z-Score** | `(current - mean) / std > 2` | 用量波动大 |
| **增量归因** | `delta > baseline + slack` | 持续监控型 |

## 4. ROI 计算公式

### 4.1 单项优化 ROI

```
ROI = (节省金额 - 实施成本) / 实施成本 × 100%

其中：
- 节省金额 = (当前成本 - 优化后成本) × 优化后运行月数
- 实施成本 = 人力工时 × 时薪 + 风险成本
```

### 4.2 综合优化 ROI（年度）

```
年度 ROI = Σ(单项 ROI × 实施概率) / N
```

### 4.3 优化建议输出模板

```markdown
## 优化建议 #N：[类型] - [资源ID]

- **资源**：cvm-xxxxxx（生产环境，2C4G，按量计费，CPU < 5% × 7 天）
- **当前成本**：¥45/天 = ¥1,350/月
- **优化后成本**：包月 ¥800/月（降 40.7%）
- **风险等级**：中（需业务确认停机时间）
- **审批 owner**：业务方
- **实施成本**：约 0.5 人天
- **预期年化节省**：¥6,600
- **回本周期**：< 1 个月
- **实施步骤**：
  1. 业务方确认可停机时间
  2. 创建 CVM 镜像备份
  3. 转包月/降配
  4. 验证业务
  5. 监控 7 天无异常后归档旧实例
```

## 5. 预算制定建议

### 5.1 预算公式

```
建议预算 = max(过去3月均值, 去年同期) × (1 + 业务增长率) × 1.1
```

**业务增长率参考**：
| 业务阶段 | 增长率 |
|---|---|
| 起步期（< 6 月）| +30-50% |
| 增长期（6-24 月）| +15-30% |
| 稳定期（> 24 月）| +5-15% |
| 收缩期 | -10% - 0% |

### 5.2 预算分级

| 级别 | 阈值 | 动作 |
|---|---|---|
| 警戒 | 80% | 通知 owner，准备 review |
| 告警 | 100% | 通知 owner + 财务，强制 review |
| 超支 | 120% | 冻结新增资源（可选） |

### 5.3 预算分摊

按 Tag 维度（业务/部门/项目）拆分预算：
```yaml
budgets:
  - name: "电商业务线月度预算"
    amount: 50000
    tag_filter: { business-line: "电商" }
  - name: "数据库月度预算"
    amount: 20000
    product_filter: ["cdb", "redis", "mongodb", "postgres"]
```
